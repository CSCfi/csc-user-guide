## Overview

The SD Desktop service has been upgraded to a new version and the changes are not fully backward comaptible. This guide explains the actions you need to take to continue accessing your virtual desktop and to restore import/export functionality using the new Data Gateway application. 

Your next steps depend on whether your virtual desktop currently has a volume attached.

## Step 2: Choose the Correct Option for Your Needs

| Scenario | Recommended Action | Advantages | Disadvantages |
|----------|------------------|------------|---------------|
| Virtual desktop **has a volume** and you want minimal effort | Stay on the current virtual desktop and import the new data gateway | - No migration required<br>- Keep using your existing environment without reinstalling the software<br>- Quick and simple solution | - No access to new features<br>- Cannot attach/detach volumes while the virtual desktop is running<br>- Limited future compatibility<br>- Cannot attach multiple volumes to the same virtual desktop<br>- All project members must import the new data gateway<br>- |
| Virtual desktop **has a volume** and you plan to continue the analysis long-term | Move the volume to a new virtual desktop | - Access to all new features<br>- Data gateway is available and up to date by default<br>- Ability to manage volumes while the desktop is running<br>- Supports attaching multiple volumes<br>- Better long-term solution <br>- fllu security patched | - Requires manual work<br>- Requires reinstalling the software<br>- Migration effort and team coordination needed |
| Virtual desktop **has no volume** | Move to a new virtual desktop | - Clean and stable environment<br>- Avoid risk of system issues<br>- Access to the latest features | - Requires exporting all files - Requires setting up a new environment |
