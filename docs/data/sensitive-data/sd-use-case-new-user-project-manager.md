## Käyttötapaus { #use-case }

Tarvitset pääsyn SD-palveluihin tutkimusdatan käsittelyä varten. Olitpa tutkimusryhmän vetäjä tai työskentelet itsenäisesti, voit käyttää SD Connectia tutkimusdatan tallentamiseen, jakamiseen ja siirtämiseen. Halutessanne tiimisi voi myös analysoida SD Connectiin tallennettua dataa SD Desktopin avulla.

## Vaiheittainen ohje { #step-by-step-tutorial }

### 1. Luo CSC-tunnus { #1-create-a-csc-account }

- **Siirry [MyCSC-portaaliin](https://my.csc.fi){ target="_blank" }**
- Kirjaudu sisään Virtulla tai Hakalla kotiorganisaatiosi federaation perusteella. Valitse kotiorganisaatiosi ja kirjaudu sen tunnistautumispalveluun. [Kuinka saada tili ilman Hakaa tai Virtua](../../accounts/how-to-create-new-user-account.md#getting-an-account-without-haka-or-virtu).
- Täytä tietosi Sign up -sivulla.
- Luo salasana, jossa on vähintään 12 merkkiä, sekä isoja ja pieniä kirjaimia ja vähintään yksi numero. Erikoismerkit eivät ole sallittuja.
- Saat vahvistuksen CSC-käyttäjätilistäsi sähköpostitse.

### 2. Luo uusi CSC-projekti { #2-create-new-csc-project }

- Siirry **Projects**-sivulle (vasen sivuvalikko tai hampurilaisvalikko oikeassa yläkulmassa).
- Valitse sivun yläosasta **New project**
- Täytä projektin nimi ja projektikuvaus. Voit muokata näitä myöhemmin tarvittaessa.
- Valitse **Project category** -kenttään **Academic** (jos olet tutkija ja suomalaisen korkeakoulun jäsen).
- Jos projektissasi käsitellään henkilötietoja, valitse kenttään **We handle personal data in this project** arvo "Yes".
- Varmista seuraavaksi käyttöehtojen alla, että täytät **Prerequisites and Responsibilities** -kohdan vaatimukset CSC-projektipäällikölle. Tutkimusprojekteissa projektipäällikön tulisi olla kokenut tutkija (esim. postdoc, ryhmänjohtaja, professori tai tutkimusorganisaation palveluksessa oleva tohtorikoulutettava). **Huomio opiskelijat:** Jos olet opiskelija, pyydä ohjaajaasi luomaan CSC-projekti tai [ota yhteyttä CSC Service Deskiin](../../support/contact.md) (aihe: sensitive data). SD-palveluihin ei voi päästä opiskelijatunnuksella.
- Lue ja hyväksy käyttöehdot.
- Klikkaa **Create new project**.

![Luo uusi projekti.](https://a3s.fi/docs-files/sensitive-data/MyCSC/MyCSC_NewProject_Academic.png)

### 3. Tietosuoja { #3-data-protection }

#### 3.1 Käsittelytoimien kuvaus (GDPR) -lomake { #3-1-description-of-processing-activity-gdpr-form }

Kun käytät projektiasi ensimmäistä kertaa ja projektissa käsitellään henkilötietoja, sinua pyydetään täyttämään Käsittelytoimien kuvaus (GDPR) -lomake. Lomakkeessa kuvataan keskeiset tiedot, kuten kerättävän datan tyypit (esim. arkaluonteiset henkilötiedot), käyttötarkoitus ja tietoturva sekä rekisterinpitäjä (yleensä kotiorganisaatiosi). Lomake liitetään CSC-projektiisi ja ohjaa CSC:tä datan käsittelyssä. Voit muokata dokumenttia myöhemmin tarvittaessa.

#### 3.1 Henkilötietojen siirto ETA-alueen ulkopuolelle { #3-1-data-transfer-outside-the-eea }

Kun siirrät henkilötietoja tai arkaluonteista dataa ETA-alueen ulkopuolelle, varmista, että siirrolle on asianmukainen oikeusperuste ja että noudatat soveltuvaa tietosuojalainsäädäntöä. Huomioi tämä jakaessasi arkaluonteista dataa yhteistyökumppaneille SD Connectin tai SD Desktopin kautta.

#### 3.2 Ota yhteyttä organisaatiosi lakipalveluihin { #3-2-contact-your-organization-s-legal-office }

Jos tarvitset apua edellä mainituissa asioissa tai tukea sen varmistamiseen, että SD-palvelut soveltuvat tutkimusdatanne käsittelyyn, ota yhteyttä organisaatiosi [tietosuojavastaavaan tai lakipalveluihin](../../accounts/when-your-project-handles-personal-data.md#contact-information-of-finnish-universities-data-protection-legal-offices). Voit toimittaa heille esimerkiksi seuraavat tukimateriaalit:

1. [Technical and Organizational Measures (TOMS)](../../data/sensitive-data/technical-organisational-sec-measures.pdf)
2. Palvelukuvaukset: [SD Connect](https://research.csc.fi/-/sd-connect) ja [SD Desktop](https://research.csc.fi/en/-/sd-desktop)
3. [CSC:n henkilötietojen käsittelysopimus (DPA)](https://research.csc.fi/data-processing-agreement)
4. GDPR-lomake (Käsittelytoimien kuvaus), jonka voi ladata omasta CSC-projektistasi
5. [CSC:n tietopolitiikka](https://www.csc.fi/en/data-policy)
6. [Service Level Agreement](../../data/sensitive-data/sd-services-sla.pdf)

![Käsittelytoimien kuvaus](https://a3s.fi/docs-files/sensitive-data/MyCSC/MyCSC_Description.png)

### 4. Aktivoi SD-palvelut projektille { #4-activate-sd-services-for-your-project }

- Palvelut lisätään projektisivun oikean alakulman **Services**-ikkunasta. Klikkaa **Add services**.
- Valitse **SD Connect**. Allas lisätään automaattisesti, sillä se on taustalla oleva tallennusratkaisu.
- Valitse **SD Desktop**.
- Klikkaa **Add selected services**.

![Lisää uusia palveluja -painike.](https://a3s.fi/docs-files/sensitive-data/MyCSC/MyCSC_AddServices1.png)

### 5. Hae Billing Units -resursseja projektille { #5-apply-billing-units-for-your-project }

#### Jokainen aktiivinen CSC-projekti kuluttaa sekä Billing Units -yksiköitä (BU) että tallennuskiintiötä { #every-active-csc-project-consumes-both-billing-units-bus-and-storage-quota }

1. **Billing Units (BU:t)** käytetään seuraamaan, kuinka paljon laskentatehoa ja käsittelyaikaa projektisi kuluttaa.

CSC käyttää eri tyyppisiä Billing Units -yksiköitä palvelusta riippuen:

- SD Desktop kuluttaa **Cloud Billing Units** -tyyppiä, joka kuvaa virtuaalityöpöytien ja laskentaresurssien käyttöä.
  
- SD Connect kuluttaa **Storage Billing Units** -tyyppiä, joka kuvaa CSC:n pilvi-infrastruktuuriin tallennetun datan määrää.

Jokainen uusi projekti alkaa **3 000** Cloud- ja Storage BU:lla. Tämä riittää yleensä alkuvaiheen testaukseen, mutta sinun tulee suunnitella, kuinka paljon BU:ita tarvitset **1 vuodeksi**.

**Voit hakea lisää BU:ita CSC-projektiisi MyCSC-portaalissa**:

- Kirjaudu [MyCSC:hen](https://my.csc.fi), avaa CSC-projektisi. Uudessa näkymässä selaa alas ja siirry **Resources**-ikkunaan.

  Klikkaa **Apply for resources** ja seuraa vaiheittaisia ohjeita.

1. **Kiintiö** rajoittaa käytettävissä olevan tallennustilan määrää. Oletuskiintiö (eli tallennustila) kullekin CSC-projektille, jossa SD Connect on käytössä, on **10 TB**. Sitä voidaan laajentaa jopa 200 TB:iin ottamalla yhteyttä Service Deskiin _(aihe: Increase Allas quota)_.

#### SD Connectin BU-kulutus { #sd-connect-bu-consumption }

SD Connect tallentaa datan CSC:n Allas-infrastruktuuriin ja tarjoaa lisäksi automaattisen salauksen ja salausavainten hallinnan. SD Connect kuluttaa Billing Units -yksiköitä nopeudella **1,3 Storage BU:ta per TB per tunti**.

Alla on taulukko, joka tiivistää, kuinka paljon Storage Billing Units -yksiköitä tarvitaan datan tallentamiseen **SD Connectissa** **6 kuukauden** ja **1 vuoden** ajalta eri tallennusmäärille:

| Tallennusmäärä | Storage Billing Units (6 kk) | Storage Billing Units (1 vuosi) |
|------------------|------------------------------|----------------------------|
| 500 GB (0,5 TB) | 2 847 yksikköä | 5 694 yksikköä |
| 1 TB | 5 694 yksikköä | 11 388 yksikköä |
| 10 TB | 56 940 yksikköä | 113 880 yksikköä |
| 100 TB | 569 400 yksikköä | 1 138 800 yksikköä |

#### SD Desktopin BU-kulutus { #sd-desktop-bu-consumption }

Datan analysointi SD Desktopissa kuluttaa **Cloud Billing Units** -yksiköitä käytetyn virtuaalityöpöydän tyypin mukaan. Jokaisella virtuaalityöpöydällä voi olla myös volume (levytila), jonne voidaan tuoda tiedostoja SD Connectista/SD Applysta. Volumen voi lisätä vain työpöytää luotaessa (enintään 200 GB). Jos tarvitset suuremman volumen, ota yhteyttä Service Deskiin _(aihe: Sensitive data)_.

Alla on taulukko, joka tiivistää, kuinka paljon Cloud Billing Units -yksiköitä tarvitaan **SD Desktopin** käyttöön **6 kuukauden** ja **1 vuoden** ajalta eri työpöytävaihtoehdoilla:

| Työpöytävaihtoehto | Cloud Billing -kulutus (yks./h) | Cloud Billing Units (6 kk) | Cloud Billing Units (1 vuosi) |
|----------------|---------------------------|--------------------------|------------------------|
| Small Computation | 5,2 | 22 464 | 44 928 |
| Medium Computation | 10,4 | 44 928 | 89 856 |
| Heavy Computation | 65 | 280 800 | 561 600 |
| Small GPU Computation | 78 | 341 640 | 683 748 |
| Big Picture -projekti | 195 | 854 100 | 1 708 770 |

![Klikkaa Apply for resources.](https://a3s.fi/docs-files/sensitive-data/MyCSC/MyCSC_AddResources1.png)

#### Mitä tapahtuu, jos projektisi Billing Units -yksiköt loppuvat? { #what-happens-if-your-project-runs-out-of-billing-units }

Kun CSC-projektisi kaikki Cloud Billing Units -yksiköt on käytetty, **pääsy SD Desktop -palveluun rajoitetaan**. Tämä tarkoittaa, että:

- kaikki parhaillaan käynnissä olevat virtuaalityöpöydät keskeytetään automaattisesti
- et voi käyttää virtuaalityöpöytien sisältöä tai jatkaa niiden suorittamista ennen kuin olet hakenut lisää resursseja
- datasi säilyy muuttumattomana eikä sitä poisteta, vaikka projektin kokonais-BU:t laskisivat nollaan tai negatiivisiksi.

!!! Note
    Jokainen projektin jäsen saa MyCSC-portaalista sähköposti-ilmoituksen, kun projektin Billing Units -yksiköt ovat loppumassa.

### 6. Jos sinulla on tutkimusryhmä, lisää heidät projektisi jäseniksi { #6-if-you-have-a-research-team-add-them-to-be-your-project-members }

- Siirry projektisivulle ja valitse oikea projekti (vasen sivuvalikko tai hampurilaisvalikko).
- Lisää jäseniä **Members**-ikkunasta. Klikkaa **Add members**.
- Uudessa ikkunassa voit etsiä ja lisätä jäseniä omasta organisaatiostasi.
- Jos tarvitset jäseniä muista organisaatioista, voit luoda kutsulinkin klikkaamalla **Invitation link**.
    - Generoi linkki ja käytä sitä sähköpostissa tai verkkosivulla.
    - Henkilöt voivat hakea projektisi jäsenyyttä klikkaamalla kutsulinkkiä.
    - Tämän jälkeen sinun on hyväksyttävä heidät projektisi jäseniksi MyCSC:ssä **Members**-ikkunan **Membership applications** -välilehdellä.

![Lisää projektin jäseniä -painike.](https://a3s.fi/docs-files/sensitive-data/MyCSC/MyCSC_AddMembers1.png)

![Etsi ja lisää jäseniä.](https://a3s.fi/docs-files/sensitive-data/MyCSC/MyCSC_AddMembers2.png)

![Luo kutsulinkki.](https://a3s.fi/docs-files/sensitive-data/MyCSC/MyCSC_AddMembers3.png)

!!! Note
    Jokaisen CSC-projektin jäsenen on myös luotava MyCSC-käyttäjätili (tämä tapahtuu, kun he hakevat projektisi jäsenyyttä) ja aktivoitava MFA.

### 7. Projektin päättäminen ja tietojen säilytys { #7-closing-your-project-and-data-retention }

CSC-projektin oletuskesto on yksi vuosi, minkä jälkeen projekti suljetaan automaattisesti ja kaikki data poistetaan. Projektipäällikkönä voit joko pidentää projektin kestoa tai sulkea projektisi heti tutkimuksen päätyttyä. Projektipäällikkönä sinun tulisi tutustua siihen, miten [CSC-projekti suljetaan](../../accounts/how-to-manage-your-project.md#project-closure) MyCSC-portaalissa.

- Saat sähköposti-ilmoituksen MyCSC-portaalista, kun projektisi on vanhenemassa. Pidä yhteystietosi ajan tasalla MyCSC:ssä, jotta varmistat ilmoituksen saamisen.
- Projektin sulkemisen jälkeen pääsy SD Connect- ja SD Desktop -palveluihin estetään.
- Kaikki SD Connectiin tai SD Desktopiin tallennettu data, mukaan lukien tiedostot, virtuaalityöpöydät ja levyt, **poistetaan pysyvästi 90 päivän kuluttua projektin sulkemisesta/vanhenemisesta**.

!!! Note
    Kun data poistetaan CSC:n tietojen säilytyskäytännön mukaisesti, sitä ei voida palauttaa eikä palautumista voida tehdä.

![Sulje projekti.](https://a3s.fi/docs-files/sensitive-data/MyCSC/MyCSC_CloseProject.png)
